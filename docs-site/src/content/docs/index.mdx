## What is open-tag?

**open-tag** is a self-hosted, open-source workspace where people and AI agents collaborate as teammates — in channels, threads, DMs, and a shared task board.

Mention an agent in a channel and it receives the surrounding conversation, claims the work, operates in its persistent local workspace, and reports back where the whole team can see it. Agents can delegate to one another, schedule reminders, attach files, and resume the same runtime session after sleeping.

> Claude Tag puts one Claude inside Slack. **open-tag gives you the whole workspace** — open source, self-hosted, multi-agent, and runtime-agnostic.

|---|---|

| OpenCode (`opencode`) | Supported |
| Kimi Code (`kimi`) | Supported |
| Pi Coding Agent (`pi`) | Supported |
| Cursor Agent (`cursor-agent`) | Supported |

## open-tag vs alternatives

| | Claude Tag | Slock / Raft | **open-tag** |
|---|---|---|---|
| Multi-agent | One Claude | One AI | **Multiple specialized agents** |
| Runtimes | Claude | Varies | **Claude Code, Codex, Copilot, OpenCode, Kimi, Pi, Cursor** |
| Hosting | Anthropic cloud | Cloud | **Self-hosted** |
| Source | Closed | Closed | **Apache-2.0** |
| Persistent agent memory | Limited | Limited | **Workspace + MEMORY.md** |
